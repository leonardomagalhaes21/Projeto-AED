{\footnotesize\ttfamily void Request\+Log\+::save (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})}



Save the request log to a file. 

@detail This function is responsible for saving the request log to a text file named "{}log.\+txt"{} located in the "{}../schedule"{} directory. The request log includes details of each operation performed, such as the action type, new and old course and class codes, the class\+\_\+\+Uc\+\_\+\+Code, and the student\textquotesingle{}s code. The function begins by opening the file for writing and checks if the file opening was successful. If the file cannot be opened, it reports an error to the standard error stream and returns without saving the log. For each operation in the log, the function writes the relevant information to the file, including the action type and associated data. After recording the operation in the log file, the {\ttfamily save\+To\+Data()} method is called on the operation to update the data as needed. Finally, the function prints a success message to the standard output.

\doxysubsubsection{\texorpdfstring{undo()}{undo()}}
{\footnotesize\ttfamily void Request\+Log\+::undo (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})}



Undoes the most recent operation in the operations list. 

@detail This function undoes the most recent operation stored in the operations list. The list of operations is implemented as a queue, where the oldest operation is at the front (position 0), and the most recent operation is at the back. The function removes the most recent operation, shifting all remaining operations towards the front of the queue.

\begin{DoxyNote}{Note}
This function will only have an effect if the operations list is not empty.
\end{DoxyNote}
Time Complexity\+: The average time complexity is O(n), where n is the number of operations in the list.
